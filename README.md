# 笔试 Ai 助手

笔试 Ai 助手是一个用于个人练习、题目复盘和多端数据同步的辅助工具。它包含 Windows 桌面端、本地网页端和 Netlify 云端网页端，支持截图、图片上传、OCR/视觉理解、模型解析、账号登录同步和模型配置同步。

> 重要说明：本项目仅用于个人学习、练习和复盘，不用于真实考试、面试作弊或绕过任何平台规则。

## 核心能力

- Windows 桌面端截图：支持区域截图、长按右键选区、全屏截图、添加本地图片和图片转文字。
- 多图题目分析：可以先连续截取多张图片，再统一发送给模型分析，适合一页里有多个题目的场景。
- 精简答案输出：默认先给出答案选项或结论，再给出必要解析，避免长篇回答影响复盘效率。
- 流式输出：模型生成时前端逐步显示内容，减少等待时的不确定感。
- 图片预览：桌面端和网页端都支持查看最新截图，网页端支持点击放大查看。
- 账号系统：通过邮箱验证码注册账号，登录后可同步当前题目、答案、截图和模型配置。
- 多端互通：同一个账号在 Windows 端、网页端和移动浏览器登录后，可以看到同步后的最新数据。
- 模型配置同步：同账号下的 API Base URL、API Key、模型名、输出长度、温度和超时时间可跨端同步。
- 云端部署：支持部署到 Netlify，云端函数负责账号、验证码、同步和模型请求代理。

## 适用场景

- 平时刷题时保存截图并让模型给出简洁解析。
- 电脑端截图后，在手机或网页端查看答案和图片。
- 多台设备使用同一个账号保存同一套模型配置。
- 复盘图形题、选择题、编程题或图片题。

运行时会生成一些本地数据文件，例如：

## 桌面端使用

直接打开根目录下的程序：

```text
PracticeDesktopAssistant_Local.exe
```

桌面端启动后会同时开启本地网页服务，默认地址：

```text
http://127.0.0.1:8786/
```

如果要在手机上访问本地网页，需要电脑和手机处于同一局域网，并使用电脑的局域网 IP，例如：

```text
http://电脑局域网IP:8786/
```

## 截图与分析流程

推荐流程：

1. 打开需要练习或复盘的题目页面。
2. 使用快捷键或桌面端按钮截取题目图片。
3. 如果一页有多个题，可以连续截取多张。
4. 按 `Ctrl + Alt` 统一发送当前截图给模型分析。
5. 查看流式输出结果，答案会优先显示选项或结论，然后再显示解析。

桌面端只保留最新截图用于同步展示，避免旧图片反复堆积。分析过的图片不会长期保留在云端历史记录里。

## 快捷键

常用快捷键：

```text
Ctrl + Alt + R      区域截图
Ctrl + Alt + H      长按选区截图
Ctrl + Alt + F      全屏截图
Ctrl + Alt + O      图片转文字
Ctrl + Shift + R    备用区域截图
Ctrl + Shift + H    备用长按选区
Ctrl + Shift + F    备用全屏截图
Ctrl + Shift + O    备用图片转文字
F8 / F9 / F10 / F11 备用热键
```

长按右键选区逻辑：

1. 长按鼠标右键约 2.5 秒，选择截图区域左上角。
2. 再次长按鼠标右键约 2.5 秒，选择截图区域右下角。
3. 程序完成区域截图，但不会立即发送模型。
4. 按 `Ctrl + Alt` 后统一发送当前截图进行分析。

全局热键已做低级键盘监听处理，即使当前焦点在浏览器、文档或其他窗口中，也可以触发桌面端截图操作。

## 账号注册与登录

账号使用邮箱注册：

1. 输入邮箱。
2. 点击发送注册验证码。
3. 到邮箱中查看验证码。
4. 输入验证码完成注册。
5. 登录时输入邮箱和密码。
6. 忘记密码时，通过邮箱验证码修改密码。

登录成功后，程序会弹出登录成功提示，并显示当前账号。后续同一个账号在桌面端、网页端和移动端登录后，可以同步数据。

## 多端同步说明

同一个账号登录后，同步的数据包括：

- 最新题目截图。
- 最新模型分析结果。
- 当前账号的模型配置。
- 当前练习状态。

同步方式：

- Windows 桌面端会把最新状态同步到云端。
- 云端网页端和移动浏览器会拉取同账号的最新状态。
- 网页端分析时会暂停自动拉取，避免旧数据覆盖正在生成的答案。

“同一个账号共用数据”不是指多端保存同一个明文 token，而是指登录后服务端识别同一个用户身份，再把该用户的数据保存到同一份账号空间中。

## 模型配置

需要配置兼容 OpenAI Chat Completions 接口的模型服务：

```text
API Base URL
API Key
模型名称
最大输出
温度
超时时间
```

示例模型名：

```text
qwen3.7-plus
```

实际可用模型取决于你的模型服务商。配置完成后可以点击测试连接。登录账号后，模型配置会随账号同步，换设备登录后无需重复配置。

## 云端版本

云端版本部署在 Netlify 后，可以通过公开网址访问。它负责：

- 邮箱验证码注册。
- 登录与密码修改。
- 账号数据同步。
- 网页端模型请求代理。
- 网页端显示桌面端同步过来的最新截图。

生产环境需要在 Netlify 后台配置环境变量。

常用环境变量：

```text
RESEND_API_KEY      Resend 邮件服务 API Key
EMAIL_FROM          发信人，例如：练习助手 <noreply@你的域名>
EMAIL_REPLY_TO      回复邮箱
AUTH_CODE_SECRET    用于验证码签名的长随机密钥
```

如果使用 Resend 发信，需要先在 Resend 添加并验证发信域名，然后在 DNS 平台添加 Resend 给出的 TXT、MX、CNAME 等记录。验证成功后，邮箱验证码才能稳定发送。

## Netlify 部署

安装依赖：

```powershell
pnpm install
```

预览部署：

```powershell
pnpm run deploy:preview
```

生产部署：

```powershell
pnpm run deploy:prod
```

Netlify 会读取：

```text
netlify.toml
static/
netlify/functions/
```

构建输出目录是 `static`，云端函数目录是 `netlify/functions`。

## 常见问题

### 为什么网页端只显示“见随附图片”？

新版已经支持把桌面端最新截图压缩后同步到云端。网页端同账号登录后，应能看到最新截图缩略图，并支持点击放大。如果看不到，先确认桌面端已经登录同一账号并完成同步。

### 为什么模型请求有时会失败？

常见原因包括模型服务商连接不稳定、网络代理切换、接口超时或远端主动断开连接。程序已加入自动重试、流式输出和超时优化，但模型服务本身仍会影响稳定性。

### VPN 会影响速度吗？

会。图形题通常需要上传图片，网络路径、延迟、丢包和模型服务所在区域都会影响耗时。是否使用 VPN 的速度差异取决于当前线路到模型服务商的质量。

### 为什么截图只保留最新一张？

这是为了避免旧截图混入下一次分析，也避免云端同步数据过大。连续多图分析时，可以在本次分析前连续截图，发送后旧图不会长期保留。

### 为什么 GitHub 没有上传 exe？

exe 是构建产物，不属于源代码。仓库默认上传完整源代码、配置和构建脚本，排除 `dist`、`build`、`node_modules`、本地运行数据和 exe。需要 exe 时可以在本地重新打包。

## 安全建议

- 不要把真实 API Key 写入 README 或提交到 Git。
- `desktop_settings.json`、`sync_store.json` 等本地运行数据不应提交。
- Netlify 环境变量应在后台配置，并勾选 Secret。
- 邮件发信域名应完成 DNS 验证后再用于生产环境。
